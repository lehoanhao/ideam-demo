## EPIC 1 — 顧客情報管理 (Customer Management)

### Feature 1.1 — 顧客検索 (Customer Search)
**User story**
- As a **sales/admin user**, I want to **search customers** by id/name/tag so that I can quickly find target accounts.

**Fields / Data**
- Customer: 営業先
- Search keys: 顧客ID, 顧客名, タグ

**Acceptance Criteria**
```gherkin
Scenario: Search customer by ID
  Given I am on the Customer Search screen
  When I search by a valid customer ID
  Then the system shows matching customers

Scenario: Search customer by name (partial match)
  Given I am on the Customer Search screen
  When I search by partial customer name
  Then the system shows customers whose names match the keyword

Scenario: Search customer by tag
  Given I am on the Customer Search screen
  When I filter by one or more tags
  Then the system shows customers that have those tags
```

---

### Feature 1.2 — 顧客の作成・編集・削除 (Create / Edit / Delete Customer)
**User story**
- As a **sales/admin user**, I want to **create, update, and delete customers** so that customer master data stays accurate.

**Fields / Data**
- Contact info: 営業先の連絡情報（複数）
- Internal owners: 社内の営業担当者（複数）
- Tags: (implied)

**Acceptance Criteria**
```gherkin
Scenario: Create a customer with multiple contact methods
  Given I am on the Customer Create screen
  When I input customer required fields and add multiple contact entries
  And I submit
  Then the customer is created and visible in search results

Scenario: Edit a customer
  Given an existing customer
  When I update customer information and save
  Then the changes are persisted and reflected in the customer detail view

Scenario: Delete a customer
  Given an existing customer
  When I delete the customer
  Then the customer is no longer returned in search results
  And the system prevents deleting if there are constraints (TBD)
```

### Feature 1.3 — 顧客の対応履歴 (Customer Timeline / Interaction History)
**User story**
- As a **sales user**, I want to view a **timeline of customer interactions** so that I understand proposal results and sales activities.

**Behavior**
- Timeline shows: 提案実績、営業活動履歴
- Links out to: 案件管理 (Case Mgmt), 営業活動管理 (Sales Activity Mgmt)

**Acceptance Criteria**
```gherkin
Scenario: View customer interaction timeline
  Given I open a customer detail page
  When I navigate to the interaction history section
  Then I see a timeline of proposals and sales activities in chronological order

Scenario: Navigate to related records
  Given I am viewing an item in the customer timeline
  When I click a linked item (case or sales activity)
  Then I am redirected to the corresponding detail screen
```
