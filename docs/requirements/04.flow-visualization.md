# Flow Visualization (Markdown)

## Option A — Mermaid Flowchart (recommended)
> You can paste this into any Mermaid-enabled viewer (GitHub, many docs tools) to visualize.

```mermaid
flowchart LR
  %% Actors
  C[顧客 / Customer]
  S[営業 / Sales]
  P[仕入れ担当 / Procurement]
  A[社長 / CEO Approver]
  M1[メーカー / Manufacturer]
  M2[メーカー / Manufacturer]
  M3[メーカー / Manufacturer]
  SYS[[新システム / New System]]
  HIST[(過去実績データ / Historical Data)]

  %% Flow
  S -- "① 顧客案件登録\n(Register case)" --> SYS
  HIST -- "過去実績参照\n(Reference)" --> SYS
  SYS -- "② 参考情報提示\n(Suggestions)" --> S
  S -- "③ 商品選定\n(Product selection)" --> SYS

  SYS -- "④ 仕入れ依頼\n(Procurement request)" --> P
  P -- "⑤ メーカー依頼\n(Request RFQ)" --> SYS

  SYS -- "⑥ FAX見積依頼\n(RFQ via FAX)" --> M1
  SYS -- "⑥ FAX見積依頼\n(RFQ via FAX)" --> M2
  SYS -- "⑥ メール見積依頼\n(RFQ via Email)" --> M3

  M1 -- "⑥ 見積返答\n(Quote response)" --> SYS
  M2 -- "⑥ 見積返答\n(Quote response)" --> SYS
  M3 -- "⑥ 見積返答\n(Quote response)" --> SYS

  P -- "⑦ 仕入れ価格決定\n(Finalize purchase price)" --> SYS
  SYS -- "⑧ 仕入れ内容確認\n(Confirm procurement details)" --> S

  A -- "⑨ 提案承認\n(Approval)" --> SYS
  S -- "⑩ 見積作成・提示\n(Create & present quote)" --> C
```

## Option B — Mermaid Sequence Diagram (message-level view)
```mermaid
sequenceDiagram
  autonumber
  actor Customer as 顧客 / Customer
  actor Sales as 営業 / Sales
  actor Proc as 仕入れ担当 / Procurement
  actor CEO as 社長 / CEO
  participant SYS as 新システム / New System
  participant HIST as 過去実績データ / Historical Data
  participant M as メーカー / Manufacturers

  Sales->>SYS: ① 顧客案件登録 (Register case)
  SYS->>HIST: Query past performance
  HIST-->>SYS: Past results
  SYS-->>Sales: ② 参考情報提示 (Suggestions)
  Sales->>SYS: ③ 商品選定 (Select products)

  SYS-->>Proc: ④ 仕入れ依頼 (Procurement request)
  Proc->>SYS: ⑤ メーカー依頼 (RFQ request setup)
  SYS->>M: ⑥ 見積依頼 (Send via FAX/Email)
  M-->>SYS: ⑥ 見積返答 (Receive quotes)

  Proc->>SYS: ⑦ 仕入れ価格決定 (Finalize)
  SYS-->>Sales: ⑧ 仕入れ内容確認 (Sales confirmation)

  CEO->>SYS: ⑨ 提案承認 (Approval)
  Sales->>Customer: ⑩ 見積作成・提示 (Send quotation)
```
