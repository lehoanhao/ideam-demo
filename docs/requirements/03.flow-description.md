# End-to-end Flow (image3)

## Actors / Components
- **顧客 (Customer)**
- **営業 (Sales)**
- **新システム (New System)**
- **過去実績データ (Historical Performance Data)**
- **仕入れ担当 (Procurement)**
- **メーカー (Manufacturers / Suppliers)**
- **社長 (CEO / Approver)**

---

## Main Flow (1 → 10)

### (1) 顧客案件登録 — Register a customer case
- **営業 → 新システム**
- Sales registers a **case/project (案件)** for a customer in the new system.

### (2) 参考情報提示 — Show reference suggestions
- **新システム → 営業**
- The system uses **過去実績データ** to provide **reference information / suggestions** to Sales.

### (3) 商品選定 — Select products
- **営業 → 新システム**
- Sales selects **products (商品)** based on system suggestions and/or manual selection.

### (4) 仕入れ依頼 — Create procurement request
- **新システム → 仕入れ担当**
- The system creates a **procurement request (仕入れ依頼)** and assigns it to Procurement.

### (5) メーカー依頼 — Request quotations from manufacturers
- **仕入れ担当 → 新システム**
- Procurement triggers **manufacturer requests (メーカー依頼)** in the system.

### (6) 見積依頼と返答 — RFQ and responses via FAX/Email
- **新システム ↔ メーカー**
- The system sends **quotation requests (見積依頼 / RFQ)** to multiple manufacturers and receives **responses (返答)** via:
  - **FAX見積依頼と返答**
  - **メール見積依頼と返答**

### (7) 仕入れ価格決定 — Decide procurement price
- **仕入れ担当 → 新システム**
- Procurement reviews responses and finalizes the **procurement price/conditions (仕入れ価格決定)**.

### (8) 仕入れ内容確認 — Sales confirms procurement details
- **新システム → 営業**
- The system presents the finalized procurement details for Sales to **review/confirm**.

### (9) 提案承認 — CEO approval
- **社長 → 新システム**
- The CEO/Approver performs **proposal approval (提案承認)** for the case (e.g., approval workflow).

### (10) 見積作成・提示 — Create and present quotation to customer
- **営業 → 顧客**
- Sales creates the customer-facing **quotation (見積)** and **presents/sends** it to the customer.
