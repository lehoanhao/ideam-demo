# Screen Spec — 提案入力 (Proposal Entry) — based on image4, image5

> Goal: Define **screen layout + fields + grid columns + behaviors** so an AI Agent can recreate a similar UI.

---

## 1) Screen Overview

- **Screen name (JP):** 提案入力
- **Screen name (EN):** Proposal Entry / Proposal Input
- **Primary users:** 営業 (Sales)
- **Related roles:** 仕入れ担当 (Procurement), 社長 (Approver) (via approval status)
- **Purpose:** Create / edit a proposal record and manage its line items (products), including request/approval statuses and dates.

---

## 2) Layout (High level)

### A. Header / Top control row
- Processing mode (処理区分)
- Responsible person (担当者)
- Customer selector (顧客)
- Proposal identifiers (ID / 提案No)
- Actions (表示, 削除)
- Key dates (作成日, 最終更新日, 決定日, 納入日)
- Check box (リストアップ計上)

### B. Approval / Budget block
- R承認, 要承認 (request approval)
- P承認, 要承認
- 予算合計
- 予算備考

### C. Classification block
- 部門
- 大分類
- 目的
- 固有名

### D. Rival block (competitor)
- ライバル (multiple dropdown inputs)
- ライバル備考 (free text)

### E. Notes + change log mini area
- 備考 (large text area)
- 行追加 date + text
- 変更 date + text

### F. Summary grid (small table above main grid)
- Columns shown: 受注日 / 品名 / 納入業者 / 納��数 / 納入単価
- Appears as a compact list (likely read-only summary)

### G. Main line-item grid (editable)
- Spreadsheet-like table with many columns (see section 4)

## 5) Behaviors / UX rules (inferred)

1. **Process Type (処理区分)** controls screen mode:
   - `1: 登録` = create new
   - `2: 修正` = edit existing
   - `3: 業務` = operational mode (TBD)
2. **Budget Total (予算合計)** is auto-calculated from line items (likely sum of `予算数 * 予算単価`).
3. **Line-item grid** supports:
   - adding multiple rows
   - per-row selection checkbox
   - many dropdown columns (e.g., 品群)
4. **RFQ sending** is not shown here, but line items include fields for RFQ request type/date and results-related fields.
